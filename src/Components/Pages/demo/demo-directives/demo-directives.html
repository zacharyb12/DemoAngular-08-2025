NgIf

La directive *ngIf
permet d’afficher ou masquer un élément HTML selon une condition.
Si la condition est vraie, l’élément s’affiche,
sinon, il est complètement retiré du DOM.
C’est une manière simple de contrôler dynamiquement ce que l’utilisateur voit à l’écran

Block else

*ngIf peut être utilisé avec else
pour afficher un autre bloc quand la condition est fausse,
grâce à un -ng-template- .
C'est utile pour alterner entre deux contenus sans répéter la logique

Voici l'informations si username ,n'est pas vide!!!!!!!!

@if

Depuis Angular v17, @if et @else
offrent une syntaxe plus lisible et moderne pour gérer des blocs conditionnels,
comme en JavaScript classique.
Cela rend le code plus clair, surtout avec plusieurs conditions

@if(username){

Si la condition est remplie

} @else {

Si la condition n'est pas remplie

}

ngSwitch est utilisé pour tester plusieurs cas possibles selon une valeur donnée.

Cela remplace efficacement une suite de *ngIf

NgFor

*ngFor sert à parcourir une liste et afficher chaque élément.
On peut itérer sur des tableaux, comme une liste de noms,
pour afficher dynamiquement leur contenu dans la page HTML

{{name}}

avec un système de track by intégré.
Cela améliore la performance et la lisibilité du code,
surtout avec des objets complexes

NgClass

ngClass permet d’ajouter ou de retirer des classes CSS dynamiquement selon des conditions.
Cela permet de styliser un élément selon l’état d'une variable (ex: erreur, succès...)

NgStyle

ngStyle permet d’appliquer des styles en ligne dynamiquement,
selon les données du composant.
C’est très utile pour adapter le style d’un élément en fonction
d’une logique ou d’une action utilisateur

NGStyle

Directive Perso

Une directive personnalisée permet d'ajouter des comportements réutilisables à des éléments HTML.
Ici, la directive Changecolor change automatiquement la couleur du texte,
sans modifier le code HTML principal
